# JavaScript Entry - Engineer Training Design

## 目的

このリポジトリは、JavaScriptを通じて
「エンジニアとしての基礎思考力」を育成することを目的とする。

学習目標は以下の通り：

- 問題を分解できる
- ロジックを構造化できる
- 状態を管理できる
- Gitで履歴を管理できる
- AIを"使われる側"ではなく"使う側"として扱える

JavaScriptは手段であり、目的ではない。

---

## 全体設計（1年間 / 月2回）

全24回構成。

### Phase 1: 基礎ロジック（1〜6回）

- 変数
- 型（number, string, boolean）
- 条件分岐
- ループ
- 配列
- 関数

※ 毎回Git commitを行う

---

### Phase 2: ブラウザ操作（7〜12回）

- DOM操作
- イベント処理
- カウンターアプリ
- タイマー
- 簡単なゲーム

目的：
「入力 → 処理 → 出力」の三段構造を理解する

---

### Phase 3: 設計力強化（13〜18回）

- オブジェクト
- 配列＋オブジェクト
- 状態管理
- ロジックの分離
- ファイル分割

目的：
「動く」から「整理されたコード」へ進化させる

---

### Phase 4: 最終制作（19〜24回）

- アプリ企画
- 仕様整理
- 実装
- バグ修正
- 公開

ここで初めてAI支援を導入する。

---

## Git運用ルール

- 基本は public repository
- Issueベースで課題を出す
- ブランチ作成 → 実装 → Pull Request → レビュー

目的は「実務フローの体験」

---

## AIの扱い方

前半フェーズではAIを使用しない。

後半で導入し、以下を学ぶ：

- AIに仕様を伝える力
- 出力コードを批評する力
- 改善点を指摘する力

AIは代替ではなく、加速装置である。

---

## この講座の思想

コードを書く人を育てるのではない。

構造を理解し、
問題を定義し、
未来の技術変化に対応できる人間を育てる。

言語は変わる。
構造は変わらない。
