# 講座設計（中1向け / 1年）

この講座は、1年・全24回（月2回）を想定して設計します。前半で基礎操作と考え方を固め、後半で実装力とレビュー力を育てる構成です。最終的には、言語が変わっても通用する学習習慣と開発手順の獲得を目標にします。

## 全体像（Phase 1〜4）

- **Phase 1（導入）**
  - Code.org を中心に、順次実行・条件分岐・繰り返しなどの基本概念を学びます。
- **Phase 2（JavaScript基礎）**
  - JavaScript の文法とブラウザ上の動作を理解し、簡単なUI操作を実装します。
- **Phase 3（Web実装）**
  - DOM 操作、イベント、状態管理を使って、小さな機能を自力で組み立てます。
- **Phase 4（発展・統合）**
  - 仕様を整理し、実装・レビュー・改善を繰り返して、成果物の品質を高めます。

## Git運用ルール

- 初期段階は、`add / commit / push` を確実に実行できることを重視します。
- 操作に慣れてきたら、`branch / PR` を使って提出・レビューの流れへ移行します。
- 課題管理は Issue を起点にし、変更履歴を追える状態を維持します。

## AI（バイブコーディング）の導入方針

- 前半は原則として AI を使用せず、手で実装して基礎理解を固めます。
- 後半で AI を導入し、以下の流れを実践します。
  - **仕様化 → 生成 → レビュー → 修正**
- AI 出力はそのまま使わず、意図の説明と検証をセットで行います。

## 言語は変わっても応用できるスキル

- **抽象化**: 共通パターンを見つけ、問題を一般化して捉える力
- **分解**: 大きな課題を小さな作業に分け、順序立てて進める力
- **状態管理**: 値の変化を追跡し、画面や処理の整合性を保つ力
- **履歴管理**: Git の履歴を使い、変更理由と経緯を再現可能にする力
